# Hate Speech Detection with Machine Learning

A machine learning project for detecting hate speech, offensive language, and neutral text using classical text mining techniques.

The project compares different text representation methods and machine learning classifiers on the **Davidson Hate Speech and Offensive Language Dataset**.

---

## Project Overview

The goal of this project is to analyze and compare several classical machine learning approaches for hate speech detection.

The task is treated as a **multi-class text classification problem** with three target classes:

| Class ID | Class Name |
|---:|---|
| 0 | hate_speech |
| 1 | offensive_language |
| 2 | neither |

The project focuses on:

- text preprocessing
- Bag of Words representation
- TF-IDF representation
- supervised machine learning classifiers
- stratified k-fold cross-validation
- per-class evaluation metrics
- analysis of overfitting and generalization

---

## Dataset

The project uses the **Davidson Hate Speech and Offensive Language Dataset**.

The dataset contains tweets labeled into three classes:

- `hate_speech`
- `offensive_language`
- `neither`

## Machine Learning Approach

The project compares two text representation methods:

| Representation | Description |
|---|---|
| Bag of Words | Represents text using word occurrence counts |
| TF-IDF | Represents text using term frequency and inverse document frequency |

The following classifiers are evaluated:

| Model | Notes |
|---|---|
| Complement Naive Bayes | Naive Bayes variant suitable for imbalanced text classification |
| Logistic Regression | Uses `class_weight="balanced"` |
| Random Forest | Uses `class_weight="balanced"` |

Complement Naive Bayes is used as the Naive Bayes approach because it is better suited for imbalanced text classification problems than the standard Multinomial Naive Bayes.

---

## Evaluation Strategy

The models are evaluated using **Stratified K-Fold Cross-Validation**.

The following values of `k` are tested:

```text
k = 3, 5, 10
```

This results in:

```text
3 models × 2 text representations × 3 k values = 18 experiments
```

Stratified K-Fold is used because the dataset is imbalanced and the class distribution should be preserved across the folds.

---

## Evaluation Metrics

The project uses both global and per-class metrics.

### Global Metrics

| Metric | Purpose |
|---|---|
| Accuracy | Overall percentage of correct predictions |
| Precision Macro | Average precision across all classes |
| Recall Macro | Average recall across all classes |
| F1 Macro | Main metric for comparing models |
| F1 Weighted | F1-score weighted by class frequency |

### Per-Class Metrics

For each class, the following metrics are calculated:

- Precision
- Recall
- F1-score

This allows a more detailed analysis of how well the models recognize each class individually.

Special attention is given to the `hate_speech` class because it is the minority class and is more difficult to classify correctly.

## Text Preprocessing

The preprocessing stage applies the following operations:

- converts text to lowercase
- unescapes HTML entities
- removes URLs
- removes user mentions
- keeps hashtag words but removes the `#` symbol
- removes retweet markers
- removes punctuation, digits, and special symbols
- removes extra spaces

Example:

```text
Original:
RT @user: This is an example tweet!!! #Example

Cleaned:
this is an example tweet example
```

## Experimental Design

Each experiment is defined by a combination of:

- one text representation method
- one classifier
- one value of `k` for Stratified K-Fold Cross-Validation

Example experiment:

```text
Vectorizer: Bag of Words
Model: Complement Naive Bayes
K: 5
```

The same evaluation procedure is applied to all models in order to make the comparison fair and consistent.

---

## Main Results

The final report ranks the models by **Validation Macro F1-score**.

The most important columns in the report are:

| Column | Meaning |
|---|---|
| `F1 Macro` | Main metric for overall model comparison |
| `Hate Recall` | How many real hate speech examples were correctly detected |
| `Hate F1` | Balance between hate speech precision and recall |
| `F1 Gap` | Difference between train and validation Macro F1 |

The `F1 Gap` is used to analyze possible overfitting:

```text
F1 Gap = Train F1 Macro - Validation F1 Macro
```

A larger gap may indicate that the model performs much better on training data than on validation data.

---

## Why Complement Naive Bayes?

Complement Naive Bayes was selected as the Naive Bayes approach because it is designed to work better with imbalanced text datasets.

Unlike standard Multinomial Naive Bayes, Complement Naive Bayes uses statistics from the complement of each class, meaning all examples that do not belong to the current class.

This makes it more stable when some classes are much larger than others.

In this project, this is important because the `hate_speech` class is significantly smaller than the `offensive_language` class.
